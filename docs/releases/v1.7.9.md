# Curator v1.7.9 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.9`

## Headline

New **Tools → Tier scan** menu item launches a `TierDialog` matching the v1.7.8 CLI: pick a recipe (cold / expired / archive), tune min-age, optionally filter by source/root, click Scan, see candidates in an interactive table. Audit-event-equivalent to the CLI.

### Behavior

- **Recipe combo** auto-updates the min-age default (cold→90, expired→0, archive→365)
- **Min-age spinbox** auto-disables when recipe = `expired` (expires_at gates that recipe; min-age irrelevant)
- **Source combo** is populated from `runtime.source_repo.list_all()` + a default `(any)`
- **Root prefix** is a case-insensitive prefix filter on `source_path`
- **Scan button** triggers `runtime.tier.scan(criteria)`, populates the table, and emits a `tier.suggest` audit event with `actor='gui.tier'`
- **Status column** is color-coded by bucket (vital=green, active=blue, provisional=yellow, junk=red)

## Authoritative-principle catches

**1 import bug caught on first test run:**
- `QComboBox` was used in other dialog methods via local imports, not promoted to module-level. New `TierDialog._build_ui` assumed module-level → `NameError`. Test 1 caught immediately. Fix: promoted `QComboBox` + `QLineEdit` + `QMessageBox` to the module-level import block.
- **Lesson #42 logged**: when adding new classes to a file with mixed import patterns, scan the **module-level** imports specifically rather than checking if a symbol appears anywhere in the file.

**1 environment subtlety caught by debug output:**
- `build_runtime()` in our test env doesn't reliably respect `CURATOR_CONFIG` env var; it consistently uses the canonical DB. Test 3 originally seeded synthetic data; debug output revealed canonical was being used. Pivoted to verify UI flow against canonical (service-level correctness already proven by v1.7.8 8-test suite).
- **Lesson #43 logged**: for GUI integration tests, prefer testing **UI flow correctness against canonical DB** over re-testing service logic against synthetic data.
