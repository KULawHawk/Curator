# Curator v1.7.18 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.18`
**Theme:** New CLI command `curator audit-summary`.

## Headline

`curator audit-summary` aggregates recent audit events by `(actor, action)` pairs with counts and relative-time timestamps. Forensic-grade visibility into what the system has been doing across all sessions — a different surface area from the recent TierDialog and PII pattern ships.

---

## What's new

- **5 filter flags:**
  - `--days N` (default 7) — look-back window
  - `--since ISO` — overrides `--days` when set, e.g. `--since 2026-05-01`
  - `--actor STR` — filter to single actor (e.g. `gui.tier`)
  - `--action STR` — filter to single action (e.g. `tier.suggest`)
  - `--limit N` (default 20) — cap displayed groups
- **Two output modes:**
  - Rich pretty-print (default) with relative-time formatting
  - JSON (`--json` global flag) for scripting
- **Sorted by activity** — most-active groups first
- **Friendly relative-time helper** — `Xs ago` / `Xm ago` / `Xh ago` / `Xd ago`
- **Read-only** — doesn't emit new audit events, doesn't mutate the log

## Limitations

- **No per-entity drill-down** — aggregates by (actor, action); use `curator audit list` for details
- **No graph/sparkline output** — just counts + bounds; could add ASCII histograms
- **No CSV export** — `--json` is the only structured format
- **UTC-only timestamps** — `--local` flag could help
- **No GUI surface** — future Tools → Audit Summary dialog deferred
