# v1.7.68 — Hoist inline ANSI-strip to shared `strip_ansi` fixture

**Date:** 2026-05-12
**Type:** DRY refactor (test infrastructure)
**Scope:** `tests/conftest.py` + 3 test files

## TL;DR

v1.7.62 rush-fixed the Rich/Typer help-output assertion problem by inlining the same ANSI-strip regex in 3 test files. v1.7.68 hoists it into a shared `strip_ansi` pytest fixture. **Pure refactor: zero behavior change, 100% pass parity.**

## The change

Before (in 3 test files):
```python
import re
output = re.sub(r"\x1b\[[0-9;]*m", "", result.output)
```

After:
```python
def test_help(self, runner, strip_ansi):
    output = strip_ansi(result.output)
```

The `strip_ansi` fixture in `tests/conftest.py` returns a closure over a module-level compiled regex pattern. Standard pytest idiom.

## Fixture design

```python
_ANSI_ESCAPE_PATTERN = re.compile(r"\x1b\[[0-9;]*m")

@pytest.fixture
def strip_ansi():
    def _strip(text: str) -> str:
        return _ANSI_ESCAPE_PATTERN.sub("", text)
    return _strip
```

- Module-level compile (once per session, not per call)
- Closure returning callable (canonical for parameterized fixtures)
- `re` import added to conftest.py top imports

## Catches

- **Rush-fix DRY violations age into permanent debt.** v1.7.62 was correct (close arc); v1.7.68 closes the debt 6 ships later.
- **Fixture > module-level function.** Sidesteps `from tests.conftest import` brittleness without forcing `__init__.py` changes.
- **Module-level pattern compile.** Avoids per-call regex compilation cost.
- **Two-step refactor**: add fixture first, then remove inline regex. Tests green at every intermediate state.
- **One test instantiates CliRunner inline.** Adding `strip_ansi` to its signature works regardless.

## What this does NOT do

- Doesn't audit perf/ or property/ test directories
- Doesn't add a pre-commit lint for new inline ANSI regex
- Doesn't add a unit test for `strip_ansi` (trivially correct)
- Doesn't expose as module-level helper (fixture-only)
- Doesn't generalize to other escape codes (none in use)

## Lesson reinforced

**Hoist duplicated patterns to shared helpers when they appear in 3+ places.** Standard software engineering hygiene; conftest.py is the right home for cross-test helpers in pytest.
