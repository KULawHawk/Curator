## Key design decisions

| Decision | Reasoning |
|---|---|
| Full 9-cell matrix (not hybrid) | Diagnostic completeness > cost; first run of cross-platform needs all info |
| `fail-fast: false` | One failure must not mask the other 8 cells |
| Job names include both dimensions | Each cell is a distinct GitHub check |
| Linux Qt deps step (`libegl1`, `libgl1`, etc.) | PySide6 offscreen plugin needs system libs on Ubuntu |
| `runner.os == 'Linux'` not `matrix.os == 'ubuntu-latest'` | Canonical pattern; survives matrix-label changes |
| Coverage artifact: `coverage-${os}-py${pyver}` | Avoids collision across 9 cells |
| No pre-emptive cross-platform fixes | v1.7.42 pattern: push, observe, fix in follow-ups |

## Authoritative-principle catches

- **`fail-fast: false` doubled.** Same principle as v1.7.50, on 2 dimensions.
- **`runner.os` over `matrix.os`** for Linux-only step. Survives label changes.
- **Coverage artifact prefix preserved.** Was `coverage-py3.11`; now `coverage-ubuntu-latest-py3.11` (kept the `py` prefix to avoid breaking downstream parsing).
- **macOS runners are now arm64.** GitHub migrated in 2025. PySide6 6.6+ has wheels; some transitive deps may not.
- **Deliberately no pre-emptive fixes.** Push first; let CI surface what actually breaks.

## Limitations

- No ARM Linux (Raspberry Pi etc.) — GitHub free runners don't include it
- No Python 3.14 yet (beta)
- No Windows 10/11 distinction (`windows-latest` = Server 2022)
- No macOS-13 or older distinction (`macos-latest` = Sonoma)
- Initial run likely has failures (expected; this is the gap-finder)
- CI minutes 3x'd (~36 → ~108/run; still within free tier)
