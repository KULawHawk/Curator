# v1.7.46 — Recycle-bin 8.3 short-path expansion (closes CI failure #4)

**Date:** 2026-05-11
**Type:** CI-caught bug fix (Class B from CI run #3)
**Scope:** `src/curator/_vendored/send2trash/win/recycle_bin.py` (+60), `tests/integration/test_recycle_bin.py` (+120, 7 tests)

## TL;DR

Adds `_to_long_path` helper that calls Win32 `GetLongPathNameW` to expand 8.3 short-path components like `RUNNER~1` → `runneradmin`. `_normalize_for_compare` now routes through this helper, so recycle-bin lookups by short path match the long-path form stored in `$I` index files. Closes the fourth and final CI failure from v1.7.44's CI run #3.

## Why it matters

GitHub Actions Windows runners have username `runneradmin` (>8 chars), so NTFS generates 8.3 alias `RUNNER~1`. `tempfile.gettempdir()` returns the short form; the Recycle Bin's `$I` index files store the long form. Without normalization, the substring match fails.

## The fix

Two-pass `GetLongPathNameW`:
1. Try the full path (works if every component exists on disk)
2. Fall back to parent-dir translation when the leaf doesn't exist (the typical post-trash case)

Returns input unchanged on non-Windows, ctypes errors, or paths with no short components. Never raises.

## Authoritative-principle catches

- **Leaf-doesn't-exist fallback is the hot path.** Initial draft only tried the full path; would have failed on the actual trash-lookup case where the leaf is gone. The parent-translation fallback fixes this and is the path the CI failure actually exercises.
- **Best-effort over exception propagation.** ctypes can fail subtly; the function catches all exceptions and returns input unchanged.
- **8.3 might be disabled on some volumes.** Modern NTFS installs sometimes disable short-name generation. On those volumes, the function is correctly a no-op.

## Limitations

- Volume needs 8.3 enabled for fallback to do anything useful
- Per-component only (multi-level short-path nesting not handled)
- No unit tests for actual short-path expansion (needs admin to synthesize)
- `_to_long_path` is underscore-prefixed (internal)
