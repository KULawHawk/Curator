## Authoritative-principle catches

- **Two test bugs caught in first run** (16/23 initial). (1) Asserted r²=1.0 but got 0.9998 because variable month lengths (Jan/Mar/May=31, Feb=28, Apr=30) make day-offset x-axis non-evenly-spaced even when cumulative GB grows by an exact constant. Relaxed to `r² > 0.99`. (2) Schema-by-memory: used `id` column but actual schema is `curator_id TEXT PRIMARY KEY`. Live-queried via `SELECT sql FROM sqlite_master WHERE type='table' AND name='files'`.
- **FK constraint requires source-first.** `files.source_id REFERENCES sources(source_id)`. Helper INSERT OR IGNOREs a source before any files.
- **Patch at import site, not source.** `curator.services.forecast.psutil.disk_usage`, not `psutil.disk_usage`.
- **1 unmissed statement accepted.** Line 296 is a fit_ok eta_99 edge case requiring pathological fixtures.
- **strftime('%Y-%m', seen_at)** requires ISO datetime strings — helper uses `f"{month}-15T12:00:00"`.

## Lesson #64 (new)

When seeding test DB rows directly via `db.execute()`, verify the schema before writing the INSERT. Column names drift across migrations and don't always match dataclass field names. A 30-second live schema query saves debugging multiple failing tests at once. Applies whenever conftest provides a migrated DB and tests bypass repositories.
