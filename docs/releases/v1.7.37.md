### New helper: `build_csv_writer(stream, dialect="csv")`

In `src/curator/cli/util.py`. Centralizes CSV writer construction:

```python
def build_csv_writer(stream, dialect: str = "csv"):
    """Build a csv.writer with the requested dialect."""
    import csv as _csv
    if dialect not in ("csv", "tsv"):
        raise ValueError(f"unknown csv dialect: {dialect!r}. Valid: 'csv', 'tsv'")
    delimiter = "\t" if dialect == "tsv" else ","
    return _csv.writer(stream, delimiter=delimiter, lineterminator="\n")
```

Every CSV/TSV-writing CLI command now calls this. The helper owns:
- The v1.7.36 `lineterminator='\n'` fix (Windows blank-line bug prevention)
- The v1.7.37 dialect/delimiter switching
- Validation of dialect strings

Future ships that need to extend CSV behavior (custom quoting, escape characters, encoding) modify one place instead of 10.

### Usage examples

```bash
# awk pipeline — TSV is easier than CSV for awk's default
curator audit --actor admin --csv --csv-dialect tsv \
    | awk -F'\t' '$4 ~ /^migration\./ {print $2, $4}'

# Spreadsheet import — some tools prefer TSV
curator sources list --csv --csv-dialect tsv > sources.tsv

# Postgres bulk-load
curator audit-export --to /tmp/audit.tsv --format tsv
psql -c "\COPY audit_archive FROM '/tmp/audit.tsv' DELIMITER E'\t' CSV HEADER"

# Back-compat: default is still CSV
curator audit --csv > audit.csv  # RFC 4180 comma-separated, unchanged
```

## Authoritative-principle catches

### Catch: wrong flag name in audit-export tests

First test run had 2 failures:
```
audit-export --out export.tsv --format tsv
# Error: No such option: --out (Possible options: --format, --to)
```

The audit-export command's output flag is `--to` (added in v1.7.31 with the semantic "export TO this path"). My test mistakenly used `--out`. Fix: 2-character edit in 2 tests. Caught in 5 seconds by typer's helpful error message.

### Design choice: helper consolidation in cli/util.py

Instead of inlining the `delimiter=` switch at 10 call sites, introduced `build_csv_writer()` in `cli/util.py`. Benefits:

1. **Single owner for CSV semantics**: v1.7.36's lineterminator fix and v1.7.37's dialect parameter both live in one place
2. **Easy future extension**: adding `--csv-quote-all`, custom escape chars, or encoding overrides means modifying one function, not 10 sites
3. **Testable in isolation**: 7 of the 13 v1.7.37 tests are pure unit tests of the helper, no CLI subprocess overhead
4. **Pattern consistency**: matches the v1.7.30+ approach of centralizing CLI utilities in `cli/util.py` alongside the glyph constants

The helper is also format-agnostic at the stream level — it accepts any writable text-mode stream, so audit-export's file-write path uses the same helper as stdout-writers.

### No new lesson this ship

The implementation was clean throughout. v1.7.36's lineterminator fix carried forward naturally; the dialect parameter slotted in cleanly. The only catch was a typo-level test bug. Lessons captured in earlier ships continue to apply (#46–#56).

## Limitations

- **Invalid dialect produces a Python traceback.** `build_csv_writer` raises `ValueError`, but CLI commands don't catch it before invoking the writer. Output is functional but ugly. Fix in v1.7.38: wrap each `if csv_output:` branch with try/except, or validate upfront via a typer Option callback. ~10–20 lines.

- **No semicolon, pipe, or arbitrary-delimiter dialects.** Only `csv` and `tsv`. Adding `;` (European Excel) or `|` (some log formats) requires a `--csv-delimiter` flag accepting an arbitrary character. Not exposed today.

- **No QUOTING control.** All output uses Python `csv` module's `QUOTE_MINIMAL` (the default). Tools needing `QUOTE_ALL` (e.g. for tools that hate unquoted dates) must post-process.

- **No encoding override.** UTF-8 only, matching the rest of Curator. Tools expecting Latin-1 or other encodings must transcode externally (`iconv`, etc.).

- **GUI doesn't expose dialect picker.** The GUI uses programmatic API access for data display, not subprocess CSV. Adding a dialect picker to GUI export dialogs is out of scope.

- **`--csv-dialect` accepts arbitrary strings at the typer layer.** No `Enum`-backed validation. Caught at the helper layer (deeper than ideal — see first limitation). Tradeoff: keeps option declaration simple at all 9 sites.
