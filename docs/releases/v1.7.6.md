## Headline

New `curator scan-pii` command detects **SSN, credit card, US phone, and email** patterns via regex with severity-graded reporting and last-4-char redaction. Detect-only baseline; downstream routing/quarantine hooks deferred until false-positive rates are measured on real data.

---

## What's new

### Service layer: `services/pii_scanner.py` (~290 lines)

```python
from curator.services.pii_scanner import PIIScanner, PIISeverity

scanner = PIIScanner()  # head_bytes=2MB by default

# 3 entry points
report = scanner.scan_text("SSN: 123-45-6789")            # one string
report = scanner.scan_file("/path/to/file.txt")           # one file (capped)
reports = scanner.scan_directory("/path", extensions=[".txt"])  # tree walk

# Per-report API
report.match_count        # int
report.has_high_severity  # bool (SSN or credit_card hit)
report.by_pattern()       # {"ssn": 2, "email": 1, ...}
report.truncated          # bool (file exceeded head_bytes)

# Per-match API
for m in report.matches:
    print(m.pattern_name, m.severity, m.redacted, m.line, m.offset)
```

### 4 default patterns

| Pattern | Severity | Regex |
|---|---|---|
| `ssn` | HIGH | `\b\d{3}-\d{2}-\d{4}\b` |
| `credit_card` | HIGH | `\b(?:\d[ -]?){13,16}\b` (no Luhn yet) |
| `phone_us` | MEDIUM | covers `(XXX) XXX-XXXX`, `XXX-XXX-XXXX`, `XXX.XXX.XXXX`, `XXXXXXXXXX`, with optional `+1` |
| `email` | MEDIUM | `\b[\w.+-]+@[\w-]+(?:\.[\w-]+)+\b` |

### CLI: `curator scan-pii <path>`

```
$ curator scan-pii ./client_data --high-only --show-matches

PII scan results
  Target:           ./client_data
  Files scanned:    127
  Files with PII:   12
  HIGH severity:    3
  Total matches:    47

Per-file findings:
  ./client_data/intake_2026-01.txt
    email=4, phone_us=2, ssn=1
      L  12           ssn  *******6789
      L  45         email  *********.com
      ...
```

Flags:
- `--recursive / --no-recursive` (default: recursive)
- `--ext .EXT` (repeatable filter)
- `--head-bytes N` (override 2 MB cap)
- `--show-matches` (print individual redacted matches)
- `--high-only` (filter to SSN / credit_card hits)
- `--json` (machine-readable output)

- **11-test headless suite** against synthetic + temp files:
  1. SSN detection (HIGH severity, last-4 redaction)
  2. Credit card (13-16 digits with separators)
  3. Phone (4 formats: parens, dashes, dots, plain)

## Limitations / next steps

- **No Luhn validation** on credit card matches — would cut ~10x false positives but is its own mini-feature; defer
- **No organize integration** — detect-only baseline. A v1.8 hookspec will gate migration destinations on PII status
- **No semantic detection** — names-in-context, institution-specific MRN formats are T-D02 territory
- **No GUI integration** — CLI-only for now; a Tools-menu "Scan for PII" dialog is a v1.8 polish
- **UTF-16 / non-UTF-8 encodings** decode via `errors='replace'` — a UTF-16 file with PII could yield 0 matches if every other byte is null. Edge case; not observed in practice.
