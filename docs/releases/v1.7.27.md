# Curator v1.7.27 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.27`
**Theme:** TierDialog bulk migrate — GUI counterpart to v1.7.25's CLI `--apply`.

---

## Headline

TierDialog gains a **"Migrate Selected..." button** in the footer plus a **"Migrate to..." context menu entry**. The GUI now has feature parity with v1.7.25's CLI `tier --apply --target`. The tier story is symmetric across both interfaces.

---

## Workflow

**Bulk migrate (footer button):**
1. Open TierDialog (Tools menu → Tier scan)
2. Pick recipe, set Root prefix, click Scan
3. Select candidates in the table (Ctrl+click / Shift+click for multi-select)
4. Click "Migrate Selected..."
5. Pick a target directory in the file picker
6. Confirm the count + size
7. Done — files moved, table refreshes, audit events logged

**Single-file migrate (context menu):**
1. Right-click a row → "Migrate to..."
2. Same target picker, same confirmation, single-row migration

---

## What's new

### New methods

- **`_get_selected_file_entities()`** — helper that maps the table's selected rows to FileEntity objects via the existing `_resolve_row_to_file_entity` helper. Forgiving: drops rows where resolution fails.

- **`_action_bulk_migrate(file_entities)`** — core workflow. Mirrors CLI semantics:
  - Validates non-empty selection + Root prefix set
  - `QFileDialog.getExistingDirectory()` for target
  - `QMessageBox.question` confirmation
  - `MigrationService.plan()` from `root_prefix` → `target`
  - Filter moves to selected `curator_id`s
  - Audit-brackets: `tier.apply.start` + `tier.apply.complete`
  - `MigrationService.apply(include_caution=True)`
  - Tally outcomes + show result dialog
  - Refresh scan to drop migrated files from the table

### New UI

- **"Migrate Selected..." footer button** — between the v1.7.17 hint label and the Close button
- **"Migrate to..." context menu entry** — below "Send to trash..."
- **Audit actor split**: GUI bulk migrates emit with `actor='gui.tier'`; CLI applies emit with `actor='cli.tier'`. Dashboards can now distinguish CLI- vs GUI-driven migrations.

## Safety design

Identical to v1.7.25:

| Guarantee | How |
|---|---|
| Root prefix required | Warning dialog with example path if missing |
| Native target picker | `QFileDialog.getExistingDirectory()` |
| Confirmation by default | `Yes \| No` modal, default `No` |
| `include_caution=True` | Tier recipe IS the safety signal |
| Empty selection | Friendly info dialog with multi-select hint |
| Scan refresh after | Migrated files disappear; remaining stay |

## Authoritative-principle this turn

**0 bugs caught** — clean first-try ship.

**Design lessons reinforced:**

1. **Audit actor strings distinguish entry points** — `cli.tier` vs `gui.tier`. Dashboards can answer "how is the team using tier?" with concrete data.

2. **Headless Qt testing via monkeypatched modals** — class-level monkeypatch of `QFileDialog.getExistingDirectory` / `QMessageBox.question` / `QMessageBox.exec` for the duration of the test, restore in `finally`. Works without ever rendering a dialog.

3. **Single-row context menu → list of one** — no separate code path needed. `_action_bulk_migrate([file_ent])` reuses the bulk method. Same validation, same audit trail.

4. **Programmatic table selection in tests** — `QItemSelection` + `QItemSelectionModel.SelectionFlag.Select` simulates user multi-select deterministically without input events.

## Limitations

- **No `--dry-run` equivalent in GUI** — could add a "Preview..." button
- **No `--keep-source` equivalent** — hardcoded MOVE mode; could add a checkbox in the confirm dialog
- **No progress indicator for large migrations** — synchronous `apply()` freezes the UI. Future: switch to `create_job()` + `run_job()` with `on_progress` callback + `QProgressDialog`
- **No Ctrl+M keyboard shortcut** — mouse-only entry; trivial to add via existing `eventFilter`
- **Whole scan refresh after migrate** — re-runs full scan instead of incrementally removing migrated rows. Wasteful for 10k+ candidates
