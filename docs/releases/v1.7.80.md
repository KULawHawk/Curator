## The 17 principles

**Building (I-VII):**
1. Empirical CI > theoretical reasoning
2. Conservative defaults are options, not commitments
3. Functional parity > code parity (cross-platform)
4. Minimal-scope credentials force good infrastructure
5. Defense-in-depth via fallbacks
6. TTY-aware output (Unix convention)
7. POSIX-canonical paths align with host conventions

**Correctness (VIII-XI):**
8. Pre-commit lints turn invariants into laws
9. Bug-class sweeps + regression lints prevent recurrence
10. Pre-push hooks are signals, not gates
11. DRY refactors pair with regression lints

**Communication (XII-XV):**
12. Documentation follows tooling, not leads it
13. Workflow files accumulate decision history in comments
14. Audit closure has value even when result is "no action"
15. Backlog items deferred 3+ times must be closed or scheduled

**Automation (XVI-XVII):**
16. One-command setup is non-negotiable
17. Dependabot is the change-detector, you're the change-acceptor
