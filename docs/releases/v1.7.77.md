# v1.7.77 — Accept Dependabot PR #1: checkout v5→v6, upload-artifact v6→v7

**Date:** 2026-05-12
**Type:** Decision-and-bump ship (validates v1.7.71 Dependabot design)
**Scope:** `.github/workflows/test.yml`

## TL;DR

Dependabot's first grouped PR (PR #1) proposed bumping checkout v5→v6 and upload-artifact v6→v7. **CI ran 9/9 GREEN on the bump branch (fa16240)**, empirically validating that v1.7.67's conservative v5 choice was unnecessary. This ship lands the bumps directly.

## The empirical test

v1.7.67's conservative reasoning:
> "v6 has breaking changes around credential persistence we don't need"

Dependabot's PR #1 tested it. Result: ✅ 9/9 GREEN on v6+v7.

**Why v6 is safe for Curator:**
Curator's CI: `checkout → setup-python → install → pytest → upload coverage`. No step after checkout uses credentials, submodules, or push. The v6 credential-persistence change is invisible to this workflow.

## What this ship does

1. Bumps `actions/checkout` v5 → v6
2. Bumps `actions/upload-artifact` v6 → v7
3. Updates workflow header comment with decision history
4. Documents reasoning in CHANGELOG and these release notes

PR #1 itself stays open in the GitHub UI (PAT lacks `pull_requests:write`); Jake can close manually, or Dependabot auto-closes when it detects no diff remains.

## Why manual bump, not API merge

v1.7.74 recommends a minimal-scope PAT (`actions:read` only). Merging PRs needs `pull_requests:write`, which would broaden token scope. Manual bump preserves the minimal-scope policy and achieves the same end state.
