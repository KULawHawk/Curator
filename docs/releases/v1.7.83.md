## Apex principle, codified

**Accuracy is THE apex principle.** Coverage standard:
1. Ship at 100% line + branch coverage, OR
2. Mark `# pragma: no cover` on specific lines with a documented justification

Dead defensive code should be **removed**, not tested or pragma-marked. "Diminishing returns" is corner-cutting language with no place in an accuracy-first project.

## Lessons captured

**Lesson #71 — ACCURACY IS THE APEX PRINCIPLE.**
I shipped v1.7.81 at 99% and v1.7.82 at 94% with a "good enough" framing. Jake pushed back: *"why would we not ship once we've hit 100%? you forget accuracy is the apex principle."* He was right. Every uncovered branch is real production code that real users can hit. The cost of one more test is bounded; the cost of an untested production branch firing in the field is not. Going forward: 100% line + branch is the default ship bar; `# pragma: no cover` with justification is the only exception.

**Lesson #72 — Dead defensive code is worse than testing it.**
The `if parent:` check guarded against an unreachable case (`str(Path(...).parent)` is never empty). A `# pragma: no cover` would technically hit 100%, but the dead check would mislead future readers into thinking an edge case exists. **Rule: when a defensive guard is statically unreachable, remove it. Reserve `# pragma: no cover` for genuinely defensive code where the impossibility is contextual** (e.g. "this requires pydantic validation to have been bypassed").

**Lesson #73 — Stub-fidelity bugs only surface under accuracy-mandate testing.**
The original `StubFileRepository.find_candidates_by_size` returned all files of matching size including deleted ones — the real SQL has `WHERE deleted_at IS NULL`. The gap was invisible at 94% coverage; it only became a test failure when `test_fuzzy_index_returns_deleted_file_skipped` was written to chase the 100% bar. **The 100% bar isn't just about lines hit — it's a forcing function that surfaces fidelity gaps in the test infrastructure itself.** This is the second time Lesson #70 has fired; the meta-lesson is that the 100% bar is what makes Lesson #70 violations visible.
