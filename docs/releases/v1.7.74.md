# v1.7.74 — Auto-install dev setup script: codify hooksPath + PAT manual steps

**Date:** 2026-05-12
**Type:** Developer-onboarding tooling (PowerShell installer)
**Scope:** `scripts/setup_dev_hooks.ps1`

## TL;DR

v1.7.70 noted: "no auto-install script that sets up hooksPath, PAT file, and ci_diag.ps1 in one go." **This ship adds `scripts/setup_dev_hooks.ps1`** — an idempotent installer that:
1. Sets `git config core.hooksPath .githooks`
2. Creates `~/.curator/github_pat` (with secure prompt + format validation)
3. Verifies the setup and prints quick-reference commands

## Why this matters

By v1.7.73, a new clone needed: 2 git hooks + 3 lint tests + PAT file + diag script. Each step trivial; cumulative friction real. This ship reduces it to one command.

## What it does

**Step 1: hooksPath**
- Detects existing config; skips if already `.githooks`
- Verifies `.githooks/pre-commit` and `.githooks/pre-push` exist

**Step 2: PAT setup** (skippable with `-SkipPat`)
- Ensures `~/.curator/` exists
- Skips if `github_pat` file already exists with valid prefix
- Otherwise: prompts via `SecureString` with step-by-step PAT creation guide
- Marks file hidden (best-effort)

**Step 3: Verify** — re-reads config, lists what each hook does, prints quick-reference

## Sample output (already-configured)

```
==> Configuring git core.hooksPath...
    SKIP: core.hooksPath already set to .githooks
    OK: Pre-commit + pre-push hooks found and activated
==> Skipping PAT setup (--SkipPat)
==> Verifying setup...
    OK: core.hooksPath: .githooks
    OK: Pre-commit hook present (runs 3 lints: glyph, ORDER BY, ANSI regex)
    OK: Pre-push hook present (warns when CI is red)

Setup complete.
```

## Catches

- **Idempotent from the start** — explicit skip paths for all configured state
- **SecureString prompt** — token doesn't appear in PowerShell history
- **Prefix validation** — catches typos without API call
- **Skip-don't-overwrite** for existing PAT file
- **Explicit `-SkipPat` flag** — better than silent skip on missing token
- **Repo-root resolution from `$PSCommandPath`** — script works from any CWD
- **Color-coded output** — cyan/green/yellow/red visual hierarchy
- **In-prompt PAT instructions** — no need to leave the script

## What this does NOT do

- No bash/sh variant (PowerShell-only for now)
- No auto-revoke/rotate of PATs
- No API validation (prefix-only)
- No README documentation reference yet
- No Python venv setup (separate concern)

## Lessons reinforced

- **Per-clone setup friction accumulates silently.** Many small steps deserve an installer.
- **Idempotent installers are the right default.** No destructive overwrites.
- **Auto-installers should report skipped steps explicitly.**
