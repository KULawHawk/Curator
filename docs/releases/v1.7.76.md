# v1.7.76 — Bash variant of setup_dev_hooks for cross-platform contributors

**Date:** 2026-05-12
**Type:** Cross-platform tooling
**Scope:** `scripts/setup_dev_hooks.sh` + README

## TL;DR

v1.7.74 shipped PowerShell installer; v1.7.76 adds a functionally-identical bash variant for macOS/Linux/WSL/Git-Bash contributors. README's Contributing section shows both invocations side by side.

## Feature parity verified (15 features)

All features of the PowerShell installer reproduced in bash, using host-shell idioms:
- `stty -echo` instead of `Read-Host -AsSecureString` for hidden PAT input
- `chmod 600` instead of Hidden attribute for PAT file permissions
- `chmod +x` on the hooks (POSIX needs this; Windows doesn't)
- ANSI escape codes with `[ -t 1 ]` TTY detection
- `--skip-pat`, `--token`, `--help` flags matching PowerShell `-SkipPat`, `-Token`

## Live test passed

Ran on Git Bash for Windows: idempotent skip behavior verified, color output correct, exit code 0.

## Catches

- **Functional parity, not code parity** — two clean idiomatic implementations
- **TTY-aware color output** — ANSI when interactive, plain when piped
- **chmod 600 for PAT** — POSIX-canonical credential file protection
- **chmod +x for hooks** — POSIX requires executable bit; PowerShell skips this
- **stty wrapped with `|| true`** — defensive layering for rare missing-stty case
- **Help text extracted from script header** — single source of truth
- **README shows both invocations side by side** — visually parallel

## What this does NOT do

- No bash variant of `ci_diag.ps1` (future ship)
- No code sharing between PowerShell/bash (intentional)
- No `--non-interactive` mode beyond `--skip-pat`
- No `setup_dev_hooks.cmd` for legacy cmd.exe

## Lessons reinforced

- **Functional parity > code parity.**
- **Cross-platform tooling needs cross-platform documentation.**
- **Pre-existing POSIX assets (hooks already in `sh`) made the installer translation easier.**
