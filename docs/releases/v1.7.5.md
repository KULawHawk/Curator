## Headline

The Lineage Graph tab gains a **time-slider that replays how lineage evolved**. Drag the slider, watch edges appear chronologically. Click ▶ Play and the slider auto-sweeps from earliest to latest at 5 steps/sec.

Captures "Card VIII v1 → v2 → v3.1 → final" as an animated graph.

---

## What's new

### Time-slider UI

Above the Lineage Graph view, a new row contains:

```
Time: [================|=========]  as of: 2026-05-04 18:00  [▶ Play] [Show all]
```

- **Slider** (0–100 with tick marks every 10): drag to filter edges by `detected_at <= cutoff`
- **Time label**: monospace; updates live as the slider moves
- **▶ Play / ⏸ Pause**: animates the slider at 200ms intervals (5 steps/sec), auto-stops at 100%
- **Show all**: resets to no filter

### Behavior

- Slider value 0% → earliest `detected_at` in the DB
- Slider value 100% → latest (or all edges visible if at 100, meaning "no filter")
- Linear interpolation in between
- When the DB has 0 lineage edges (current canonical state), slider + Play disable with tooltip; tab still renders cleanly

### API surface (programmatic use)

```python
# Builder gains 2 methods + 1 keyword arg
builder.build_full_graph(max_detected_at=datetime(2026, 5, 3))  # filter
earliest, latest = builder.get_time_range()                     # bounds

# View persists filter across argument-less refresh()
view.refresh(max_detected_at=cutoff)
view.refresh()                  # preserves cutoff
view.clear_time_filter()        # reset

# GraphEdge dataclass adds detected_at field (backward-compat default=None)
```

## Authoritative-principle catch rate

1 type-coercion bug caught:
- `SELECT MIN/MAX(detected_at)` returns SQLite **strings**, not datetimes (`PARSE_DECLTYPES` not set on this connection). Test 3 caught this before the GUI's `slider_value - min_dt` arithmetic would have crashed. Fixed via `_coerce_datetime` helper that handles None / datetime / ISO-string inputs.

## Limitations / next steps

- **No history axis labels.** The slider shows current cutoff via the time-label widget, but there's no axis showing marked dates (e.g. "May 1 | May 5 | now"). A `QGraphicsItem` axis layer would be v1.8 polish.
- **No per-edge fade-in animation.** Edges appear/disappear instantly. Smooth opacity transitions would require a tween system.
- **No focus + time combination.** The focus-graph mode (in the builder but not yet wired in the tab) doesn't yet integrate with the time slider.
- **No edge count history chart.** A small inset chart showing edges-vs-time would help orient the user; deferred.
