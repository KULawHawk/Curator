# v1.7.69 — Linux `/var` audit: defensive subdivision mirroring v1.7.63 macOS fix

**Date:** 2026-05-12
**Type:** Defensive cross-OS audit (path classification)
**Scope:** `src/curator/services/safety.py`

## TL;DR

v1.7.63 surgically subdivided macOS's bare `Path("/private")` after a real CI failure (TMPDIR lives under `/private/var/folders`). Linux had the same architectural problem: bare `Path("/var")`. FHS 3.0 designates `/var/tmp` as user-writable. **This ship subdivides Linux's `/var` into 8 specific system-managed subdirs** — proactive defensive fix mirroring v1.7.63.

## The architectural problem

`/var` hosts BOTH system-managed and user-writable data per FHS 3.0:

**System-managed (correctly OS_MANAGED):**
- `/var/log`, `/var/lib`, `/var/cache`, `/var/spool`, `/var/run`, `/var/mail`, `/var/db`, `/var/empty`

**User-writable (should NOT be OS_MANAGED):**
- `/var/tmp` (FHS §5.15: user temp, persists across reboots)
- `/var/local` (FHS §5.9: site-local additions, typically writable)

Bare `Path("/var")` treats them all uniformly. Wrong for user-writable ones.

## The fix

Replace bare `Path("/var")` with the 8 system-managed subdirs (FHS-authoritative). Mirrors v1.7.63's `/private` → specific subdirs surgery.

## Why proactive

No current Linux test triggers this bug because:
- Linux pytest's `tmp_path` uses `/tmp`, not `/var/tmp`
- GitHub-hosted Ubuntu runners don't set `TMPDIR=/var/tmp` by default

But the architectural pattern is identical to macOS, and future CI runner images or developer environments could surface it. v1.7.66 philosophy: single-site bug fixes should consider the bug CLASS. v1.7.63 surfaced a class for cross-OS path enumeration; v1.7.69 sweeps it for Linux.

## Catches

- **v1.7.63 surfaced bug class; v1.7.69 closes it proactively.** Same philosophy as v1.7.66 (ORDER BY sweep). One bug class deserves one sweep.
- **Fix is strictly more permissive.** Cannot break existing behavior; only reclassifies REFUSE → SAFE for user-writable subdirs.
- **Mirrors v1.7.63 in structure and comments.** Cross-OS architectural symmetry.
- **8 subdirs derived from FHS 3.0**, not intuition. Authoritative source.
- **`/var/opt` excluded.** Some packages install writable content under `/var/opt/<package>`; SafetyService is advisory, not enforced. Erring permissive matches v1.7.63 philosophy.

## What this does NOT do

- Doesn't audit Windows OS-managed paths (no bare-too-broad patterns)
- Doesn't research current CI TMPDIR behavior (defensive regardless)
- Doesn't add unit test for the path subdivision (trivially correct)
- Doesn't add `/var/tmp` to app-data paths (transient, not "app data")
