## What landed

`tests/unit/test_pytest_qt_smoke.py` (NEW, 3 tests):

- **`test_qtbot_fixture_is_available`** — `qtbot` exposes `addWidget`, `mouseClick`, `waitSignal` (all callable). Defensive sanity check that the fixture is wired.
- **`test_widget_click_drives_signal`** — `qtbot.mouseClick(QPushButton, Qt.LeftButton)` drives the button's `clicked` signal inside a `with qtbot.waitSignal(...)` context. Canonical pattern Tier 2+ will use for menus, actions, dialog buttons.
- **`test_wait_signal_with_existing_qt_signal_class`** — `qtbot.waitSignal` works against a `Signal(object)` bridge (the Round 3 Tier 4 pattern from `ScanProgressBridge` etc.); `blocker.args` captures the payload.

## Why a gate ship

The three idioms above are the foundation of every Tier 2+ widget test. If pytest-qt or PySide6 ever break them in a future upgrade, this small focused file catches it cleanly. The alternative is discovering the breakage inside a 300-line `test_gui_main_window_part2_coverage.py` and not knowing which of 50 tests is the canary.

Cost: ~75 lines, 3 tests, ~0.6s run time. Permanent. Worth it.
