## Lessons captured

**Lesson #74 â€” Scope-narrowing is an accuracy enabler, not a quality regression.**
Every macOS/Linux line is something we *claimed* to validate without actually validating well. Scope-narrowing trades hypothetical coverage of unrun platforms for actual coverage of the platform that ships.

**Lesson #75 â€” Set aside, don't delete. Document the resume path.**
Pragma + resume doc preserves institutional knowledge; deletion forces rewrite. The rule: when narrowing scope, mark with pragma + reference a resume doc; only delete when code is genuinely wrong.

**Lesson #76 â€” Doctrine amendments should follow a stable pattern, not a single ship.**
I almost codified the 100% standard in v1.7.83 right after Jake's pushback. Principle 12 said wait. The wait was justified: safety.py's complexity (psutil mocking, platform branches, defensive paths) confirmed the bar is achievable on real-world modules. Codifying after one ship is a guess; after three is a pattern.

**Lesson #77 â€” Platform code is testable if you treat `sys.platform` as data.**
`monkeypatch.setattr(safety.sys, "platform", "darwin")` makes platform branches testable on any host. Same trick works for the unmockable: `monkeypatch.setattr(builtins, "__import__", fake_import)` forces ImportError. Any branch that depends on a module-global or import-time fact is testable with monkeypatch, given enough patience.

**Lesson #78 â€” Monkeypatching standard library methods is high-blast-radius.**
Twice during this ship, a defensive test for one branch crashed because the monkeypatched method couldn't handle a call site I didn't anticipate. The fix: selective-replacement patterns (`selective_resolve`, `selective_exists`) that fall through to the original for inputs you don't care about. Default to surgical monkeypatching, not blanket replacement.
