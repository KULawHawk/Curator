## Headline

`curator scan-pii` gains 3 more HIGH-severity API key patterns:
- **Twilio Account SID** (`AC` + 32 lowercase hex)
- **Mailgun API key** (all 3 variants: `key-`, `private-`, `pubkey-`)
- **Discord bot token** (3-segment dot format, M/N prefix)

The exact set v1.7.11 release notes flagged as deferred. Total patterns: **14** (was 11).

---

## What's new

3 new default patterns:

| Name | Severity | Regex |
|---|---|---|
| `twilio_account_sid` | HIGH | `\bAC[a-f0-9]{32}\b` |
| `mailgun_api_key` | HIGH | `\b(?:key\|private\|pubkey)-[a-zA-Z0-9]{32}\b` |
| `discord_bot_token` | HIGH | `\b[MN][A-Za-z0-9_\-]{23,30}\.[A-Za-z0-9_\-]{6,7}\.[A-Za-z0-9_\-]{27,}\b` |

### Design notes

- **Twilio** uses lowercase hex specifically — Twilio docs guarantee this. The `[a-f0-9]` constraint prevents collision with the AWS ASIA pattern which uses uppercase `[A-Z0-9]`. Verified by Test 6.
- **Mailgun** covers all 3 documented variants. Classic `key-` is deprecated but still in deployed configs; `private-` and `pubkey-` are current. All require exactly 32 alphanumeric chars.
- **Discord** bot tokens have 3 dot-separated segments. The leading `[MN]` constraint cuts false positives on random base64-shaped 3-segment strings (e.g. JWTs).

## Authoritative-principle catch

**1 test-data length bug caught immediately by lesson #45 (programmatic construction):**
- Test 6's first version had hand-typed Twilio SID `ACabcdef0123456789abcdef0123456789ab` — counted manually as 32 hex chars; actually 34.
- Fix: `twilio_hex = "abcdef0123" * 3 + "ab"` plus `assert len(twilio_hex) == 32` BEFORE the regex test.
- **Lesson #45 reinforced**: second consecutive ship where hand-typed regex test data had off-by-N errors. The fix is mechanical — construct programmatically + assert length first.

**0 regex bugs caught** — the 3 new patterns worked first-try once test data was correct.

## Limitations / next steps

- **No Twilio Auth Token** — 32-char hex with no distinctive prefix; high FP rate
- **No GCP service account JSON** — multi-line, structural scanner needed
- **No JWT detection** — `xxx.yyy.zzz` base64 format too broad without header parsing
- **No Atlassian / Bitbucket / GitLab tokens** — deferred until demand surfaces
- **No Conclave hookspec for custom validators** — still on v1.8 list
