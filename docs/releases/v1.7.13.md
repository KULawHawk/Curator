# Curator v1.7.13 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.13`
**Theme:** T-A02 polish — lineage time-slider axis labels.

---

## Headline

The Lineage Graph time-slider now shows **5 date labels** (YYYY-MM-DD) at 0% / 25% / 50% / 75% / 100% positions directly under the slider. Users can visually anchor slider position to actual dates instead of just "somewhere between earliest and latest."

---

## What's new

### Before / After

**Before (v1.7.5-v1.7.12):**
```
Time: [============O==========================] [Play] [Show all]
       as of: 2026-04-22 14:30
       (Slider has tick marks but no date references)
```

**After (v1.7.13):**
```
Time: [============O==========================] [Play] [Show all]
       as of: 2026-04-22 14:30
       2026-03-15  2026-03-30  2026-04-15  2026-04-30  2026-05-15
       ↑           ↑           ↑           ↑           ↑
       0%          25%         50%         75%         100%
```

### Implementation

- **5-label axis row** under the slider (`main_window.py`, +37 lines)
- Labels positioned with `QHBoxLayout` stretch=1 per label, 48/110 px outer margins to align with slider track
- Edge-aware alignment: leftmost left-aligned, rightmost right-aligned, middles centered
- Style: `color: #607D8B; font-size: 8pt` (Material slate gray; visually subordinate to slider)
- New helper `_lineage_axis_label_text(pct)` returns YYYY-MM-DD or empty string

## Limitations / next steps

- **No tick-mark synchronization** — slider's tick interval (10) is independent of the 4 inter-label gaps. A polish pass could align them.
- **Labels are static** — they show the time-range boundaries, not the moving slider position. The label above the slider (`as of: ...`) shows the live time.
- **No event-density histogram** — a future polish could draw a mini histogram above the slider showing edge-detection density per time bucket.
- **No timezone awareness** — local time without TZ suffix; fine for single-user lineage, would need clarification for collaborative review.
- **No locale-aware date format** — hardcoded YYYY-MM-DD (sortable, unambiguous); v1.8 i18n pass could honor user locale.
