# v1.7.72 — Pre-commit ORDER BY lint: codify the v1.7.66 sweep as a regression guard

**Date:** 2026-05-12
**Type:** Project-invariant lint (regression guard)
**Scope:** `tests/unit/test_repo_order_by_lint.py` + `.githooks/pre-commit`

## TL;DR

v1.7.66 swept 13 ORDER BY sites adding `, rowid` for SQLite tie-break determinism. **This ship adds a pytest-level lint enforcing that invariant**, wired into the pre-commit hook. The next contributor to add a single-timestamp `ORDER BY` to a repository module will be blocked at commit time with a remediation message.

## The lint

```python
# For each ORDER BY in src/curator/storage/repositories/:
#   - 5-line window after the match must contain `rowid`
#   - OR contain a column from KNOWN_UNIQUE_COLUMNS (curator_id, src_path)
#   - OR have an inline `# order-by-lint: <reason>` exemption
```

## Error message includes

- File and line number of each violation
- Concrete fix examples (DESC/ASC/multi-key patterns)
- Exemption mechanism documentation
- Pointer to v1.7.66 release notes

## Catches

- **Mirrors v1.7.32 lesson #50 lint pattern.** Same structure, same hook invocation, same error message style.
- **5-line window** captures multi-line SQL strings.
- **`KNOWN_UNIQUE_COLUMNS` set** for future extensibility (one-line edit).
- **Inline exemption** via `# order-by-lint: <reason>` — follows `# noqa:` convention.
- **Two lints in one pytest invocation** — one process startup, not two.
- **Test name documents the rule** — `test_every_order_by_has_deterministic_tie_breaker` self-explanatory in pytest output.

## What this does NOT do

- Doesn't add inline ANSI regex lint (separate future ship)
- Doesn't enforce in non-repository code (scope intentional)
- Doesn't auto-fix violations
- Doesn't scan migration SQL files
- Doesn't update README (error message is self-documenting)

## Lessons reinforced

- **Pytest-level lints + pre-commit hooks make project invariants mandatory.**
- **Defensive sweeps pair naturally with regression lints.** Sweep first; then prevent future violations.
- **Error messages should include the fix, not just the violation.**
