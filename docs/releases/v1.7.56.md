## Design

- **Real binary fixtures**, not mocks. Pillow for JPEG/PNG, zipfile for DOCX, pypdf for PDF.
- **Verifies output content**, not just exit codes. Re-reads stripped files and asserts metadata is gone.
- `pytest.fixture` for the stripper; `tmp_path` for all I/O.

## Authoritative-principle catches

- **One test had a bug.** First run: 35/36. My `_make_jpeg_with_exif` used bogus `exif_bytes`; Pillow wrote it but `_getexif()` returned None on read-back. Fix: use Pillow's `getexif()` API to build real EXIF. (Same lesson as v1.7.55's Google API key test.)
- **PNG text chunks via PngInfo, not raw bytes.** Documented API; matches stripper's `im.text` detection.
- **DOCX fixture doesn't need Word-openability.** Just needs the zip-container structure with the right entry names.
- **PDF fixture uses pypdf's add_metadata().** Documented API; tests re-read via PdfReader to verify removal.
- **Case-insensitive extension test.** Real-world: Windows users have mixed-case extensions from camera exports.

## Limitations

- All fixtures are synthetic; no real camera/Word/Adobe PDFs
- No performance tests
- TIFF subformat edge cases untested
- DOCX content preservation only verified for body text
- PDF page preservation only by count, not content
- forecast.py (29%) and watch.py (41%) still candidates
