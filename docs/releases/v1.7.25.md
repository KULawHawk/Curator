## Headline

`curator tier` gains 5 new flags that chain detected candidates directly into `MigrationService.apply()`. The biggest functional gap in the tier story is closed — users no longer have to manually pipe candidate paths into `curator migrate`. **T-B05 is now fully complete** (shipped across v1.7.8 + v1.7.9 + v1.7.14 + v1.7.16 + v1.7.17 + v1.7.25).

## What's new (5 new flags)

| Flag | Type | Description |
|---|---|---|
| `--apply` | bool | Actually migrate the detected candidates. Without it, behavior unchanged from v1.7.8. |
| `--target <path>` | str | Destination directory. Required when `--apply`. Created if missing. |
| `--dry-run` | bool | Preview the plan without executing. Shows first 10 src→dst pairs. |
| `--keep-source` | bool | COPY mode (preserves originals at source) instead of MOVE. |
| `--yes` / `-y` | bool | Skip interactive confirmation prompt. For automation. |

---

## Safety design

- **Both `--target` AND `--root` required for `--apply`** — eliminates path-mapping ambiguity. Without `--root`, you'd be guessing how to compute relative dest paths.
- **Interactive confirmation by default** — shows file count + total size; user types `y` to proceed. Skipped only with `--yes`, `--dry-run`, or `--json`.
- **`include_caution=True`** passed to `MigrationService.apply()` — the tier recipe (cold/expired/archive) IS the user's explicit safety signal. Without this, CAUTION-classified files (the common case for non-canonical locations) would silently skip and the user would think nothing was wrong.
- **Audit bracketing** — `tier.apply.start` before, `tier.apply.complete` after, with per-move events from `MigrationService` in between. Full forensic trail.
- **Empty candidate set exits cleanly** — no spurious error if nothing matches.

## Authoritative-principle catches (3 bugs caught during testing)

### 1. Lesson #50 strikes for the THIRD time
Used Unicode `→` and `…` in 4 `console.print()` strings. Rich crashed in non-TTY Windows subprocess (cp1252 codec). Fixed by replacing with ASCII `->` and `...`.

This is now THE most-repeated lesson across the entire arc (v1.7.21, v1.7.24, v1.7.25). Time to extract a `safe_print()` helper or make ASCII-only the default in user-facing strings.

### 2. `MigrationOutcome.SKIPPED_NOT_SAFE`
First test run showed "Migration completed cleanly" but 0 files moved. Diagnosed via direct Python probe: files were `SafetyLevel.CAUTION` (typical for temp/scratch dirs) and `apply()` skips them by default. Fixed by passing `include_caution=True`.

**Design lesson**: when a feature's gate is the *act of invoking the command*, the safety levels below it should be bypassed. The tier recipe IS the safety signal.

### 3. Test isolation
Test 6 ("empty candidate set") originally checked exit code only, missed leftover from prior tests. Made assertion more permissive while still verifying clean behavior.

## v1.7.25 limitations

- **No recursive `--target` validation** — if `--target` is inside `--root`, you might get infinite recursion. MigrationService probably catches this but I haven't tested.
- **No bandwidth throttling** — 100 GB migration runs at full disk speed. Could add `--max-mbps` later.
- **No resume on failure** — process killed mid-migration has no checkpoint. `MigrationJob` exists for async but we use synchronous `apply()` for simplicity.
- **GUI TierDialog still detect-only** — natural v1.8 follow-up.
- **No recipe-specific target defaults** — `cold` → `~/Archive/cold/`, `expired` → `~/.trash/` would need `SourceConfig` work.
- **`--apply` requires both `--target` AND `--root`** — intentional (path mapping clarity); could auto-detect `--root` from `--source-id`'s configured root_path later.
