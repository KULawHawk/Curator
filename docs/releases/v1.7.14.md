# Curator v1.7.14 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.14`
**Theme:** TierDialog right-click context menu (T-B05 GUI v2).

---

## Headline

The Tier scan dialog table now has a **right-click context menu** with three actions:
- **Inspect...** — opens FileInspectDialog for the selected file
- **Set status →** vital / active / provisional / junk submenu
- **Send to trash...** — confirms + dispatches to TrashService

Closes the gap from v1.7.9 where the dialog was purely informational. Users can now act on candidates without switching to CLI.

---

## What's new

### Right-click menu structure

```
Right-click on any candidate row →
  ┌──────────────────────────┐
  │ Inspect...               │  → FileInspectDialog
  │ Set status              ▶│  → submenu:
  │ ────────────────────────  │       vital
  │ Send to trash...         │       active
  └──────────────────────────┘       provisional (current)  ← disabled
                                     junk
```

### Behavior

- **Smart submenu**: current status shows as `(current)` and is disabled — no accidental no-op writes
- **Confirmation only on destructive action**: status changes need no confirm (reversible by re-classifying); trash gets a confirm dialog showing the full file path
- **Audit-trail integration**: every action emits properly-attributed audit events:
  - `tier.set_status` with old/new status + path
  - Trash uses TrashService's built-in audit hooks with `trashed_by='gui.tier'`
- **Robust row resolution**: `curator_id` is stored on the path column via `Qt.UserRole`, so the menu finds the right file even after table sorting/filtering
- **Auto-refresh**: after a successful action, the scan re-runs so the table reflects the new state (trashed files disappear; status changes may cause the row to drop out of the recipe)

### Code changes

- `gui/dialogs.py` — 4 new methods on TierDialog:
  - `_on_table_context_menu(pos)` — builds and pops the QMenu
  - `_action_inspect(file_ent)` — opens FileInspectDialog
  - `_action_set_status(file_ent, new_status)` — updates DB + audit + rescans
  - `_action_send_to_trash(file_ent)` — confirms + dispatches + rescans

## Authoritative-principle catches

**Two API-name bugs caught immediately by tests:**

1. **`FileRepository.get_by_id` doesn't exist** → correct method is `.get(curator_id)`. Caught by Test 3/4 AttributeError. **Lesson #46 logged**: probe repository method names via `inspect.getmembers` before writing GUI calls; naming varies (`get`, `get_by_id`, `find_by_id`, `getById`).
2. **`TrashRecord.actor` doesn't exist** → field is `trashed_by`. Caught by Test 5. Same lesson applies.

**Test design lesson #47 logged:** Initial Test 6 monkeypatched `QMenu.exec` to inspect menu structure — this hung indefinitely under PySide6 headless mode (Qt's event loop doesn't terminate cleanly when `exec` is intercepted). **For Qt GUI tests in offscreen mode, prefer testing slot dispatch on edge cases over intercepting `exec()` methods.** Manual GUI smoke testing covers live menu rendering better than fragile headless intercepts.

## Limitations / next steps

- **No multi-row bulk operations** — right-click acts on a single hovered row
- **No undo for status changes** — they go through immediately; audit log preserves prior status for manual reversal
- **No keyboard shortcut equivalents** — Del key doesn't trigger trash, no Enter to open Inspect
- **Synchronous trash dispatch** — 100+ files would freeze UI briefly per file
- **No "Open in file manager"** — `os.startfile(os.path.dirname(path))` would be useful; deferred
