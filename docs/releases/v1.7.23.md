# Curator v1.7.23 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.23`
**Theme:** Lineage slider tick-mark sync (T-A02 polish v2).

---

## Headline

The lineage time-slider's tick marks now align exactly with the 5 axis date labels shipped in v1.7.13. Tick interval changed from `10` → `25`, so 5 ticks appear at 0/25/50/75/100% — the same positions as the date labels.

---

## Before / After

**Before (v1.7.13–v1.7.22):**
```
|-|-|-|-|-|-|-|-|-|-|-|   ← 11 tick marks (interval 10)
2026-01    2026-03    2026-05    ← 5 date labels at 0/25/50/75/100%
```
Misaligned: tick marks don't correspond to specific labels.

**After (v1.7.23):**
```
|---|---|---|---|---|     ← 5 tick marks (interval 25)
2026-01  2026-02  2026-03  2026-04  2026-05    ← 5 date labels at same positions
```
Each tick sits directly above its corresponding date label.

## Authoritative-principle catch

**1 bug caught during testing**: my test imported `MainWindow` but the actual class is `CuratorMainWindow`. Caught immediately by `ImportError`. Fixed by probing `dir(curator.gui.main_window)`.

**Lesson reinforced**: even for "obvious" class names, **probe before importing in test files**. `MainWindow` is the common Qt convention but Curator chose `CuratorMainWindow` for namespace clarity. 5-second probe avoids 30-second test re-runs.

**0 implementation bugs caught** — the math (100/25 + 1 = 5 ticks) is straightforward, worked first try.

---

## Limitations

- **No tick-mark labels** — Qt's `QSlider` doesn't render labels under each tick; v1.7.13 labels are a separate `QLabel` row. Could combine into a custom widget later
- **Hardcoded 5 ticks** — if axis labels change to a different count, tick interval needs updating
- **No tick-label tooltip** — hovering over a tick doesn't show its date
