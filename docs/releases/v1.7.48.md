# v1.7.48 — Migration-abort flake fix (closes A1)

**Date:** 2026-05-11
**Type:** Test reliability / Tier 1 cleanup
**Scope:** `tests/unit/test_migration_phase2.py` (+70 lines)

## TL;DR

Eliminates the 5–10% false-fail rate on `test_abort_during_run_marks_cancelled` by replacing two timing races with deterministic synchronization. Test now passes **10/10** in stress runs at 2.0–2.4s each. Closes the last remaining Tier 1 item from the 44-item bulletproof analysis.

## The two races

| Race | Trigger | Effect |
|---|---|---|
| 1: Abort too early | Worker thread hadn't registered its abort_event when test called `abort_job` | `abort_job` silently no-ops → job runs to completion → `status='completed'` |
| 2: Abort too late | All 12 files processed within the 50ms sleep | Job finishes before abort → `status='completed'` |

## The fixes

**Race 1: deterministic event-registration wait**

New `_wait_for_abort_event_registered` static helper polls `svc._abort_events[job_id]` every 5ms up to 2s, returning when the event is registered. Replaces `time.sleep(0.05)`.

**Race 2: larger test-local library**

Replaced shared `medium_library` (12 files) with inline 300-file library. 300 × 100 bytes = ~30KB total disk — still fast (<2s) but worker can't possibly finish before abort fires.

**Bonus: sanity assertion**

`assert report.processed_count < n_files` verifies abort actually interrupted, catching the future case where someone shrinks the library back and reintroduces race 2.

## Lesson #62

> For threading-timing tests, replace `time.sleep` with deterministic synchronization on the production code's actual signals. The 50ms in the original test was a magic number that approximated "long enough" — but "long enough" depends on machine speed, system load, and what else is happening. Polling for the actual condition is bulletproof.

## Authoritative-principle catches

- **The two races are independent and both needed fixing.** Initial draft only fixed race 1. The bigger library is also necessary because a fast machine can process 12 files between event registration AND `abort_job` calls.
- **Helper uses private `svc._abort_events`.** Normally a code smell, but tests are in the same package — and the alternative (adding `is_running()` to the public API) expands the production surface for test-only needs.
- **Assertion messages identify WHICH race failed.** Faster debug next time.

## Limitations

- Helper depends on private `_abort_events` state; refactor of abort-signaling will break it
- 300 files is a heuristic — bump if future machines get fast enough
- Other `TestAbort` tests unchanged (no inter-thread timing)
- No automated grep for similar flake patterns elsewhere
