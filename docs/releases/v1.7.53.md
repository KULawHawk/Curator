# v1.7.53 — forecast.py SyntaxWarning cleanup (closes F1)

**Date:** 2026-05-12

## TL;DR

Fixes the last 4 `SyntaxWarning: invalid escape sequence '\,'` warnings, surfaced by v1.7.51's coverage instrumentation. The fix: properly escape backslashes in a docstring at `forecast.py:217`. Closes F1 — the only Python-deprecation warning in the codebase post-v1.7.47.

## The fix

```diff
-        Skips removable/optical drives. On Windows, this typically yields
-        C:\, D:\, etc. On Unix, the root ``/``.
+        Skips removable/optical drives. On Windows, this typically yields
+        ``C:\\``, ``D:\\``, etc. On Unix, the root ``/``.
```

Two improvements in one edit:
1. **Backslashes escaped (`\\` → silences SyntaxWarning)**
2. **RST backticks added (consistency with the matching docstring at L102, better Sphinx rendering)**

Runtime `__doc__` value is unchanged.

## Why this matters

PEP 626 marks invalid escape sequences as `SyntaxWarning` on Python 3.12+, and will become `SyntaxError` in a future Python release. Better to fix now than retrofit when 3.14 or 3.15 demands it. v1.7.51's coverage run was the right signal at the right time.
