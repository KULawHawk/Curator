## Headline

When a source is flagged `share_visibility="public"`, MigrationService now **auto-invokes the metadata stripper** on every successfully migrated file. EXIF, docProps, PDF metadata, and ICC profiles disappear from public-destination files automatically. **T-B07 is now fully complete** (v1.7.7 shipped the stripper as standalone CLI; v1.7.29 wires it into the migration pipeline).

---

## Forensic safety win

Mark a destination ONCE and never accidentally leak EXIF/docProps/PDF metadata to a public destination again:

```
$ curator sources config s3:incident-bucket --share-visibility public
[OK] Updated config for s3:incident-bucket:
  visibility -> public

$ curator migrate local /Cases/2026-XX-15 s3:incident-bucket /redacted/ --apply
# Migration completes; metadata auto-stripped per file.
# Audit trail records:
#   migration.autostrip.enabled  (1 event, at plan-start)
#   migration.metadata_stripped  (1 event per stripped file)
```

## What's new

### DB schema (migration 004)
```sql
ALTER TABLE sources ADD COLUMN share_visibility TEXT NOT NULL DEFAULT 'private';
CREATE INDEX IF NOT EXISTS idx_sources_share_visibility ON sources(share_visibility);
```

Purely additive. Existing rows get `'private'` (safe default).

### SourceConfig model
```python
class SourceConfig(CuratorEntity):
    # ... existing fields ...
    share_visibility: str = Field(
        default="private",
        description="'private' | 'team' | 'public'. When 'public', "
                    "MigrationService auto-strips metadata.",
    )
```

### Three valid values

| Value | Behavior |
|---|---|
| `'private'` (default) | Internal/personal. No stripping. |
| `'team'` | Shared with trusted team. No stripping (reserved). |
| `'public'` | World-readable destination. **Auto-strip enabled.** |

### CLI

```
curator sources config <id> --share-visibility {private|team|public}
```

- Validates against the allowed set (rejects bad values)
- Audited as `op=visibility`
- View-only mode now also displays `share_visibility =` line, colored green/yellow/red
- JSON view includes `"share_visibility"` key

### MigrationService auto-gating

**Constructor**: new optional kwargs `source_repo` and `metadata_stripper`, both default `None` (backward compatible).

**apply()** at the top resolves auto-strip ONCE:
```python
auto_strip = False
if (self.source_repo is not None
        and self.metadata_stripper is not None):
    dst_source = self.source_repo.get(plan.dst_source_id)
    if dst_source and dst_source.share_visibility == "public":
        auto_strip = True
        # emit migration.autostrip.enabled audit event
```

**After each successful move**:
```python
if auto_strip and move.outcome in (MOVED, COPIED,
                                    MOVED_OVERWROTE_WITH_BACKUP,
                                    MOVED_RENAMED_WITH_SUFFIX):
    self._auto_strip_metadata(move)
```

**`_auto_strip_metadata`** strips to a temp file alongside the destination, then atomic rename. Failures are caught + audited but do NOT fail the migration.

### New audit events

| Event | Emitted | Details |
|---|---|---|
| `migration.autostrip.enabled` | Once per `apply()` when auto-strip active | `plan_move_count`, source_id |
| `migration.metadata_stripped` | Per stripped file (success) | `dst_path`, `outcome`, `bytes_in`, `bytes_out`, `fields_removed` |
| `migration.metadata_strip_failed` | Per file where stripping raised | `dst_path`, `error` |

## Limitations

- **No `'team'`-specific behavior yet** вҖ” accepts but treats same as `'private'`
- **No GUI for `share_visibility`** вҖ” Sources tab could surface a dropdown
- **No per-migration `--no-autostrip` opt-out** вҖ” would need temporary visibility flip
- **No `strip_keep` allowlist** вҖ” stripper removes all metadata uniformly
- **`autostrip.enabled` emits at plan-start** even if 0 files succeed
