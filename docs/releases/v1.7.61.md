# v1.7.61 — COLUMNS=200 in CI: fixes Rich help wrapping on POSIX

**Date:** 2026-05-12
**Type:** CI infrastructure / Rich/Typer terminal-width fix
**Scope:** `.github/workflows/test.yml` (+1 env var)

## TL;DR

v1.7.60 unblocked the macOS+Linux install step; tests ran on POSIX for the **first time in project history**. Result: **1808 passed, 3 failed** on Ubuntu/3.12 (matching pattern on macOS). The 3 failures are all `'--apply' in stdout` style substring assertions against Typer's Rich-rendered `--help` output. On POSIX CI, the terminal width differs from Windows, causing Rich to wrap option names across line boundaries with embedded ANSI codes. **Fix: `COLUMNS: "200"` env var in the workflow.**

## How we found it

Jake provided a fine-grained GitHub PAT (Actions read scope). With log access, diagnosis was direct — actual failure text visible:

```
FAILED test_cli_cleanup_duplicates.py::test_duplicates_help_lists_strategies
    AssertionError: assert '--keep-under' in '<ANSI-laden stdout>'
FAILED test_cli_migrate.py::test_migrate_help_shows_phase_1_note
    AssertionError: assert '--apply' in '<ANSI-laden stdout>'
FAILED test_organize_mb_enrichment.py::test_help_lists_enrich_mb
    AssertionError: assert '--enrich-mb' in '<ANSI-laden stdout>'
3 failed, 1808 passed, 6 skipped, 10 deselected in 159.45s
```

**3 of 1817 tests.** 99.83% pass rate on the first POSIX run ever.

## Root cause

Typer's help is rendered via Rich. Rich does:
1. Box-drawing chars for the help frame (`│`, `─`, `╰`, `╯`)
2. ANSI codes for emphasis (bold flag names, dim borders)
3. **Auto-wrap on detected terminal width**

`NO_COLOR=1` (v1.7.45) disables colors but not table layout. On Windows CI, terminal width happens to leave option names intact. On POSIX CI, the wrap point falls DURING `--apply`, splitting the token with box-drawing chars + newlines + ANSI codes inserted in the middle.

## Fix

```yaml
env:
  QT_QPA_PLATFORM: offscreen
  PYTHONIOENCODING: utf-8
  NO_COLOR: "1"
  COLUMNS: "200"           # <-- v1.7.61
```

Rich honors `COLUMNS` env var for width. 200 columns = no wrapping inside option names = substring matches work.

## What this fix does NOT do
