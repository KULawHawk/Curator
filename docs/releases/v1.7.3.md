## What's new

### 4-bucket coarse taxonomy

| Bucket | Semantic |
|---|---|
| `vital` | Cannot be lost. Trash/migration veto target. |
| `active` | Default. Working files; no special treatment. |
| `provisional` | Tentative. Candidates for cleanup if not promoted. |
| `junk` | Slated for removal. Cleanup-tab targets. |

### Migration 003

Adds 3 columns to `files`:
- `status TEXT NOT NULL DEFAULT 'active'`
- `supersedes_id TEXT` (NULLable; soft UUID reference)
- `expires_at TIMESTAMP` (NULLable; retention horizon or TTL)

Plus 2 indexes: `idx_files_status`, `idx_files_expires_at` (partial NOT NULL).

Pure ALTER TABLE ADD COLUMN â€” metadata-only, no row rewrite. Applied to canonical 86,943-file DB on first run.

### `curator status` CLI subcommand group

```
curator status set <path-or-uuid> <bucket> [--expires-in-days N] [--clear-expires]
curator status get <path-or-uuid>
curator status report [--source SOURCE_ID]
```

All 3 support `--json` for machine-readable output. `status set` auto-emits an audit log entry via `cli.status` actor.
