## The helper

`src/curator/_compat/datetime.py`:

```python
def utcnow_naive() -> datetime:
    """Drop-in replacement for the deprecated datetime.utcnow()."""
    return datetime.now(timezone.utc).replace(tzinfo=None)
```

A one-line function with extensive docstring covering why it exists, why we chose naive-preservation over migrating to aware datetimes, and the future migration path.

## Why naive-preservation over timezone-aware

The modern API is `datetime.now(timezone.utc)` which returns aware datetimes. That's the right long-term direction, but converting to aware would touch:
- Naive-vs-naive comparisons throughout the codebase
- SQLite TEXT-column serialization (assumes naive UTC)
- Pydantic model fields declared as `datetime`
- GUI table display formatting
- Service-layer comparison logic

That's a v1.8.x architectural ship. v1.7.47 is the **bridge**: silence warnings now with bit-identical behavior, defer the real migration to a planned future ship.

## Authoritative-principle catches

### Catch — the 4 `default_factory=datetime.utcnow` sites

These dataclass fields take a function reference (no parens), so the first pass of the rewrite missed them. A post-pass `grep` caught them; a follow-up replaced `default_factory=datetime.utcnow` → `default_factory=utcnow_naive`. Lesson: mechanical rewrites of deprecated APIs need to audit BOTH the call form AND any reference uses (callbacks, default_factory, partial, etc.).

### Catch — `test_value_is_not_local_time` is the test I almost forgot

First-draft tests just checked "returns a datetime, is naive, is approximately now". A subtle refactor could silently change the function to `datetime.now().replace(tzinfo=None)` (local-time-as-naive instead of UTC-as-naive), which would pass all three trivial checks but be deeply wrong. The explicit UTC-vs-local discrimination test catches a future "code simplifier" that drops the `timezone.utc` argument.

### Catch — naive-preservation was the right scope

The alternative (migrate to aware everywhere) would have touched serialization, comparisons, model fields, GUI formatters. By choosing naive-preservation, this ship is mechanical, reviewable, and reversible.

## Limitations

- `utcnow_naive()` is intentionally a **temporary** helper. New code should use `datetime.now(timezone.utc)` directly. The helper will be removed when the v1.8.x aware-datetime migration lands.
- Bit-identical behavior — no behavior change, no semantics change.
- A few third-party deps (pytest_qt etc.) may still warn on `--show-warnings` runs. Out of scope.
- Python 3.14 not yet tested. Should add a CI matrix entry once it ships.
