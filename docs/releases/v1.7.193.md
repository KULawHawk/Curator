## 🐛 Bug surfaced and fixed: missing `QDialog` import

Coverage work on `_slot_source_add` revealed that `main_window.py` line 1164 references `QDialog.DialogCode.Accepted` but the **`QDialog` class was not imported** at the top of the module (imports block lines 44-57 omitted it).

**Failure mode:** any time a user successfully closed `SourceAddDialog` with the OK button, this would have raised `NameError: name 'QDialog' is not defined`.

**Likely-never-triggered-in-practice:** the path requires Tools → Sources Manager tab pivot → "Add source..." button → complete dialog → click OK. Most users probably use the CLI (`curator sources add`) instead.

**Static analysis miss:** `ruff` / `pyflakes` did not flag the missing import — likely because `QDialog` is used inside a method body, not at module scope, and Python's resolution is at lookup time.

**Fix:** one-line addition of `QDialog` to the imports list. Behavior now matches the documented contract.

**Lesson #100 / Doctrine #18 in action.** This was *not* a judgment-call deferral (the fix is mechanically unambiguous), but the partnership directive applies: speak up about what coverage work surfaces. Documenting loudly here so future code review treats the missed-import class of bugs with appropriate suspicion.

## Notable test details

- **Background-thread testing pattern** (`TestPerformMigrateResume.test_run_job_exception_in_thread_logged`) — captures the worker callable before it's wrapped in a Thread, then invokes it inline. Covers the worker's `except` branch without actually spawning a thread. Reuses the Round 3 Tier 1 sync-executor pattern (Lesson #94) for non-executor threaded code.
- **Defensive property exceptions** — `TestMigrateApplyProgressUpdate.test_get_job_id_exception_returns` replaces `_migrate_progress_model.job_id` with a raising property to exercise the `except Exception:` boundary around the property read. Same pattern as Part 1's `_RaisingConfig`.
- **`_make_source_config` sentinel fix** — initial helper used `created_at or default`, which substituted the default when `None` was explicitly passed. Switched to a sentinel-default so the production code's `if s.created_at else "?"` branch can be tested.
