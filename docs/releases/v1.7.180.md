## The deferral, finally resolved

v1.7.155 (the first ship of Round 3 Tier 3) discovered two `_resolve_file` definitions in `cli/main.py`: one at line 187 (originally introduced in v1.0.0rc1, providing UUID + exact-path + **path-prefix match**) and one at line 3721 (introduced in v1.7.3, providing only UUID + exact-path). Python binds the name to whichever `def` appears last, so the v1.0.0rc1 prefix-match feature was silently shadowed from v1.7.3 onward.

Per Lesson #100 / Doctrine #18, Code surfaced three options to Jake and did not auto-execute:
- **(a)** Delete the dead duplicate (drops the feature, drops the lie in `inspect`'s help text by updating it)
- **(b)** Merge prefix-match into the live definition (restores feature, the existing help text becomes truthful)
- **(c)** Update the duplicate's docstring only (cosmetic; doesn't address the help-text lie)

Jake chose **(b)**: the documented behavior should match reality.

## What changed

### `src/curator/cli/main.py`

- **Deleted** the dead duplicate at lines 187–225 (−39 source lines). The `# pragma: no cover` annotation added in v1.7.175 is gone with it.
- **Restored** the path-prefix match inside the live `_resolve_file`:
  ```python
  matches = rt.file_repo.query(
      FileQuery(source_path_starts_with=target, limit=2),
  )
  if len(matches) == 1:
      return matches[0]
  return None
  ```
  - `limit=2` is the clever bit: distinguishes "exactly one" (return it) from "≥ 2" (ambiguous → return None) without loading all matches.
  - Returns None on ambiguity so the caller can emit `"No file matches"` rather than auto-picking.
- **Rewrote the docstring** documenting the full lookup order and the v1.7.3 → v1.7.180 history.

## Lesson #102 captured

**Shadowed definitions become silent regressions.** When a module accumulates two top-level `def name(...):` declarations, Python binds the name to the *last* one. The first becomes silently dead. The hidden cost is not just dead lines — it's that any behavior unique to the first definition (in help text, docstrings, user-facing strings) becomes a quiet lie.

The full discovery context, application notes, and recommended Linter follow-up are in CHANGELOG.md.
