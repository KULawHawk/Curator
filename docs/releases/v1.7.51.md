## Design decisions

- **Coverage is opt-in locally, default in CI.** `pyproject.toml`'s `addopts` does NOT include `--cov`; users running `pytest` keep the fast loop. CI's workflow adds the flags explicitly.
- **`source = ["src/curator"]`** — only measure our code.
- **`branch = true`** — branch coverage in addition to line coverage.
- **`omit`** excludes `tests/`, `_vendored/`, `__main__.py`.
- **`exclude_lines`** skips `pragma: no cover`, `raise NotImplementedError`, entry guards, `TYPE_CHECKING`, Ellipsis stubs.
- **`precision = 2`** so trend tracking sees "63.79%" not "64%".
- **CI uploads `coverage.xml`** as a 30-day artifact named `coverage-py3.11` / `coverage-py3.12` / `coverage-py3.13`.
- **Codecov upload deferred** — requires repo-level secret setup; term output + XML artifact is enough signal for now.
