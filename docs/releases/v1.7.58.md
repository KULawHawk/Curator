## Lesson #65 (new)

Before `write_file` on a test file, always check `git status` first. Test files may already exist with unique tests or organizational conventions. Silently overwriting loses test history. If overwriting is intentional, preserve unique test names verbatim alongside new coverage as legacy-preserved classes.
