# v1.7.67 — GitHub Actions Node.js 24 readiness: bump checkout/setup-python/upload-artifact

**Date:** 2026-05-12
**Type:** Infrastructure modernization (workflow YAML only)
**Scope:** `.github/workflows/test.yml`

## TL;DR

GitHub deprecated Node.js 20 on 2025-09-19 with forcing date **2026-06-02** (3 weeks away) and removal date **2026-09-16**. Every CI run since v1.7.42 emitted a deprecation warning. This ship bumps all 3 actions to Node.js 24 versions, eliminating the warning and locking compatibility through 2027+.

## The bumps

| Action | Before | After | Node.js | Released |
|---|---|---|---|---|
| `actions/checkout` | `@v4` | `@v5` | 24 | Aug 2025 |
| `actions/setup-python` | `@v5` | `@v6` | 24 | Sept 2025 |
| `actions/upload-artifact` | `@v4` | `@v6` | 24 | Dec 2025 |

**Critical:** `upload-artifact@v5` only had *preliminary* Node 24 support — STILL ran on Node 20 by default. **v6** is the first version that runs on Node 24 by default. Skipping v5 entirely is the correct path.

## Why this matters

- **Forcing date 2026-06-02** — 3 weeks from now. GitHub will force-run Node 20 actions on Node 24 then. Unexpected breakages possible.
- **Removal date 2026-09-16** — Node 20 entirely removed. Actions requiring Node 20 will fail to start.
- **Proactive 3-week runway** beats reactive fire-drill on June 2nd.
- **Zero source/test changes** — pure infrastructure.

## Catches

- **Skip upload-artifact v5; jump to v6.** v5 was a half-measure (still defaulted to Node 20). GitHub's own v6 release notes are explicit on this.
- **Pick checkout v5 over v6.** Both Node 24. v5 has 9 months of production. v6 (Nov 2025) introduces credential-persistence breaking changes we don't need.
- **Caches survive.** setup-python v5 and v6 use same cache key construction (Python version + OS + path hash). No cold-cache run.
- **Artifact contract stable.** v4/v5/v6 share input names. Pure runtime change.
- **No functional risk.** YAML-only ship. CI should remain 9/9 GREEN.

## What this does NOT do

- Doesn't bump checkout to v6 (intentional — v5 is the mature choice)
- Doesn't audit sibling Ad Astra repos
- Doesn't add dependabot config
- Doesn't add pre-push lint for deprecated actions
- Doesn't migrate to OIDC
- Doesn't fix Windows live recycle-bin test (still deferred)
