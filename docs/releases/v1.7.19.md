## Headline

`curator audit-summary` gains a `--csv` flag that emits `actor,action,count,first,last` CSV instead of the Rich pretty-print table. Mirrors v1.7.18's `--json` for users who want to dump audit data into Excel / Google Sheets / Pandas.

## What's new

- **New `--csv` flag** on `audit-summary`:
  - `actor,action,count,first,last` columns
  - `count` is a bare integer (numeric in Excel)
  - `first`/`last` are ISO 8601 (parseable to datetime in Pandas)
  - Respects all existing filter flags
- **Precedence rule:** `--json` wins over `--csv` if both are set
- **stdlib `csv.writer`** for proper quoting/escaping
- **Sorted by activity** — most-active groups first

## Authoritative-principle catch

**Lesson #49 logged**: my first `Filesystem:edit_file` used the oldText `typer.echo(_json.dumps(payload, indent=2))\n return\n\n # Rich pretty-print`. **This pattern appears in 4 commands** (`scan-pii`, `export-clean`, `tier`, `audit-summary`) and the edit landed in `scan-pii` (file order). Reverted and re-did with an anchor including `period_end = datetime.utcnow()` which is unique to `audit-summary`.

**Rule**: when using line-based `Filesystem:edit_file` in a 3800+ line file with multiple similar code blocks, **the oldText anchor must include a sentinel string unique to the target function**. Generic patterns like `"# Rich pretty-print"` exist in many places; the right anchor is the first unique line *inside* the target function.

## Limitations

- **No `--csv` for other commands** — `scan-pii`, `tier`, `export-clean`, `forecast` only have `--json`; could batch as v1.8
- **No header suppression** — always emits header row; `--no-header` would help piping
- **No streaming** — full report built in memory before emit
- **No TSV** — only standard CSV; `--csv-dialect=tsv` could be added
