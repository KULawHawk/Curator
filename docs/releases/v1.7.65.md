# v1.7.65 — Celebrating the green: scripts/ci_diag.ps1 codifies lesson #67

**Date:** 2026-05-12
**Type:** Tooling addition (diagnostic helper)
**Scope:** `scripts/ci_diag.ps1` (new file)

## TL;DR

v1.7.64 closed the 20-ship CI-red arc — first 9/9 GREEN run since CI was introduced. This ship codifies the **diagnostic loop** that made the closure possible. New `scripts/ci_diag.ps1` provides one-command access to the GitHub Actions API: show all 9 cells, download logs for failing cells, or print a cross-cell failing-test summary. Lesson #67 ("diagnose with logs, not with hypotheses") called for this; this ship delivers it.

## Why this matters

Throughout the v1.7.59–v1.7.64 CI debug arc, every investigation involved ~10 manual `Invoke-RestMethod` API calls. Pattern emerged but lived as ad-hoc PowerShell incantations. This ship turns it into permanent tooling.

Next time CI goes red:
```powershell
$env:GH_TOKEN = "..."        # or ~/.curator/github_pat
.\scripts\ci_diag.ps1 status     # all 9 cells, colored output
.\scripts\ci_diag.ps1 summary    # failing tests + summaries per cell
.\scripts\ci_diag.ps1 logs ubuntu.*3.12   # deep-dive specific cell
```

## Script design

### Three modes
- `status` (default): colored 9-cell pass/fail/running grid with tally
- `logs <pattern>`: downloads logs for failing jobs matching name regex
- `summary`: extracts `FAILED tests/` lines + summary lines across all failing cells

### Token discovery (priority order)
1. `-Token` parameter
2. `$env:GH_TOKEN` env var
3. `$env:GITHUB_TOKEN` env var
4. `~/.curator/github_pat` file (single-line, 0600-style)

Last fallback enables one-time PAT setup — never paste into chat again.

### Defaults
- Repo: `KULawHawk/Curator`
- Output: `~/Desktop/AL/.curator/ci_<sha>_<jobname>.log`
