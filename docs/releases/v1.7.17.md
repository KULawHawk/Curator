# Curator v1.7.17 — Release Notes

**Tagged:** 2026-05-11
**Tag:** `v1.7.17`
**Theme:** TierDialog accelerator hints (T-B05 GUI v4).

---

## Headline

The Tier scan dialog now surfaces its keyboard shortcuts visually:
- Context menu items show **`Enter`** / **`Del`** suffixes
- A hint label in the footer reads: *"Tip: right-click for actions • Enter = inspect • Del = send to trash"*

Completes the discoverability story for v1.7.14 (right-click) + v1.7.16 (keyboard shortcuts).

---

## What's new

### Before / After

**Before (v1.7.14 + v1.7.16):**
- Right-click menu items: `Inspect...`, `Send to trash...`
- Keyboard shortcuts worked but nothing in the UI hinted at them

**After (v1.7.17):**
- Right-click menu items: `Inspect...    Enter`, `Send to trash...    Del`
- Footer label: *Tip: right-click for actions • **Enter** = inspect • **Del** = send to trash*

### Implementation details

- **Tab-separator suffix** on menu items (`"Inspect...\tEnter"`) — Qt's documented pattern for rendering right-aligned shortcut text in menu actions
- **No `setShortcut()` calls** — that would register a second binding that might fire from outside the table; the v1.7.16 `eventFilter` is the source of truth
- **Footer label** uses 8pt slate-gray (`#607D8B`) to be visually subordinate; mirrors the styling of the v1.7.13 lineage axis labels for consistency

## Authoritative-principle catch

**Lesson #47 reinforced (third occurrence):** my initial Test 2 monkeypatched `QMenu.exec` with an immediate-return function. **It still hung** under Qt's offscreen platform — apparently the event loop waits for deferred-deletion or close-event regardless of what `exec` returns.

**Replaced with `inspect.getsource(dialogs)` regex check.** Fast, deterministic, and tests what matters (the strings make it into the menu).

**Generalized lesson #47 update**: never monkeypatch `QMenu.exec` or `QDialog.exec` in headless Qt tests — even with an immediate-return function. Use `inspect.getsource` for textual content verification, or test underlying behavior via existing public hooks. The menu construction is well-typed and the action handlers are tested directly.
