## The hook in 4 lines

| CI state | Hook output |
|---|---|
| `completed / success` | Silent pass |
| `completed / failure` | Loud warning + bypass instructions |
| `in_progress` | Informational note |
| Other / no token / failure | Silent pass |

**Never blocks.** Warning only. Bypass via `git push --no-verify`.

## Token discovery (matches ci_diag.ps1)

2. `$GITHUB_TOKEN`

- **POSIX `sh`** (works on Git for Windows + mac + Linux)
- **curl preferred, wget fallback**
- **jq preferred, Python fallback** for JSON parse
- **Layered fallbacks**: every error path exits 0 (no blocking on infrastructure issues)
- **~180 lines** total
