# v1.7.73 — Pre-commit inline ANSI regex lint: codify the v1.7.68 fixture hoist as a regression guard

**Date:** 2026-05-12
**Type:** Project-invariant lint (regression guard)
**Scope:** `tests/unit/test_repo_ansi_lint.py` + `.githooks/pre-commit`

## TL;DR

v1.7.68 hoisted 3 inline `re.sub(r"\x1b\[...")` patterns into a shared `strip_ansi` fixture. **This ship adds a pytest-level lint** that blocks future inline regex regressions at commit time. Mirrors the v1.7.72 ORDER BY lint pattern. **Third project-invariant lint codified.**

## The lint

```python
# For each .py file in tests/ (excluding conftest.py and this lint file):
#   - Find lines containing the \x1b\[ regex pattern
#   - Allow inline `# ansi-lint: <reason>` exemption
#   - Otherwise: violation
```

## Catches

- **Mirrors v1.7.72 ORDER BY lint pattern** — same structure, recognizable to anyone who's seen the prior lint
- **Different exemption namespace** (`# ansi-lint:` vs `# order-by-lint:`) — no collision
- **Filename-based exemption** — simpler than path-based
- **Self-exemption is explicit** — the lint file itself contains the pattern in documentation
- **3 lints in ONE pytest invocation** — one process startup
- **Closes v1.7.68's documented limitation** — same way v1.7.72 closed v1.7.66's

## What this does NOT do

- Doesn't enforce other inline regex patterns (only `\x1b\[`)
- Doesn't scan production code
- Doesn't auto-fix violations
- Doesn't add a Windows PowerShell variant
- Doesn't update README (error message is self-documenting)

## Lessons reinforced

- **Refactor ships pair naturally with regression lints.** v1.7.68 → v1.7.73 mirrors v1.7.66 → v1.7.72.
- **Inline exemption mechanisms scale.** Each lint can have its own namespace.
- **Self-referential lints need explicit self-exemption.**
