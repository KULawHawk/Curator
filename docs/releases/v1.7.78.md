## Catches

- **Functional parity, not code parity** — same philosophy as v1.7.76
- **jq → Python fallback** — defense in depth
- **Query-specific Python fallback** — simple, not a general translator
- **POSIX-canonical paths** — `~/.curator/logs/` not Windows-style
- **`curl -sf`** — silent + fail-fast on 4xx/5xx
- **Filename sanitization** — `tr -c 'a-zA-Z0-9' '_'` matches PS regex
- **README shows both invocations side by side** — visually parallel

## What this does NOT do

- Doesn't deprecate `ci_diag.ps1` (both first-class)
- Doesn't share code between variants (intentional)
- Doesn't add `events=push` filter
- Doesn't add `--watch` mode
- Doesn't add `--workflow <name>` filter
- Doesn't generate JSON output

## Lessons reinforced

- **Cross-platform tooling needs cross-platform documentation.**
- **Defense-in-depth via fallbacks.** curl→wget, jq→Python.
- **Output paths should align with each variant's host conventions.**
