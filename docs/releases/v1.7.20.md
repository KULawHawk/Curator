## Headline

Two more `audit-summary` flag enhancements:
- **`--no-header`** ‚Äî suppresses the CSV header row (for piping/appending)
- **`--local`** ‚Äî renders timestamps in system local timezone across all output formats

Completes the v1.7.18/v1.7.19/v1.7.20 audit-summary refinement trio.

## What's new

### `--no-header`
- Suppresses `actor,action,count,first,last` CSV header
- Only meaningful with `--csv` (silently ignored otherwise)
- Use case: `curator audit-summary --csv --no-header >> existing.csv`

### `--local`
- Renders timestamps in system local TZ across all 3 output modes:
  - **Rich pretty-print**: `(local)` or `(UTC)` label after period range
  - **JSON**: adds `"timezone": "local"` field; ISO timestamps gain `+HH:MM` offset
  - **CSV**: `first`/`last` columns get TZ offset suffix
- Relative-time deltas (`"2m ago"`) are unaffected (TZ-invariant)
- **Storage unchanged** ‚Äî audit_repo still stores naive UTC; only display shifts

### Internal helper
- `_fmt_ts(dt)`: attaches `timezone.utc` to naive UTC datetimes, then `astimezone()` to local when `--local` is set

## Authoritative-principle catch

**0 bugs caught.** Clean first-try ship.

**Design note**: the correct Python-stdlib pattern for naive-UTC ‚Üí local conversion is `dt.replace(tzinfo=timezone.utc).astimezone()`. The bare `dt.astimezone()` would treat the naive datetime as local time and produce wrong results. This is documented in the `datetime` stdlib pattern; applied correctly first try.

Lesson #49 (unique-anchor edits in large files) was applied throughout: each `Filesystem:edit_file` used a sentinel string from inside the target branch.

## Limitations

- **`--local` is system TZ only** ‚Äî no `--tz America/Chicago` explicit zone control yet
- **`--no-header` doesn't apply to JSON** ‚Äî no analogous "data-only" mode
- **Relative-time labels use UTC for delta computation** ‚Äî correct behavior (TZ-invariant), but worth noting
- **Pretty-print TZ label is dim-styled** ‚Äî may be hard to see in some terminal themes
