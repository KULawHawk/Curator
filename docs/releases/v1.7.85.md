## Lessons captured

**Lesson #79 — Pure modules ship fast at 100%. Module shape determines ship cost.**

Shipping pace varied dramatically across Phase Gamma modules:
- `queries.py` (this ship): pure SQL construction → 1 pass, 0 stubs, 0.64s, 47 tests
- `tier.py`: nearly pure → 1 pass with 1 missed line (v1.7.81 → v1.7.83 to close)
- `lineage.py`: pluggy + multiple finder paths → 2 passes + source refactor (dead-code removal)
- `safety.py`: platform branches + psutil mocking + pragma decision → 2 passes + extensive monkeypatch infrastructure

**The pattern:** module complexity scales as `(pluggy?) + (platform?) + (I/O?) + (integration?)`. Pure functional modules earn fast-ship status. **Strategic implication: prefer pure modules first when picking Phase Gamma targets — they build pattern momentum that helps with the entangled ones later.** This isn't a per-ship observation; it's how to plan a Phase Gamma arc.

**Lesson #80 — Subtle default values can silently disable filters in tests.**

`FileQuery()` defaults `deleted=False` (active-only filter). My first instinct was `FileQuery()` for the "empty query" test, which would have FAILED because the default actually emits `deleted_at IS NULL` — not the empty `"1"` clause. **Rule: when testing a module with non-trivial defaults, write the empty-state test AND the default-state test as distinct tests with distinct assertions.** They cover different contracts.

**Lesson #81 — `is not None` vs falsy is a real branch class.**

`build_where()` uses `if self.min_size is not None:`. This applies the filter for `min_size=0` (a valid "any size" filter). I almost tested only with positive values, leaving the zero-case branch implicit. The explicit `test_min_size_zero_is_applied` documents the contract. **Rule: when a numeric filter uses `is not None`, write a test with value=0 specifically.** This catches future refactors that change `is not None` to a falsy check (which would silently break the zero case).
