**Tagged:** 2026-05-11

## Headline

`audit-summary` Rich pretty-print now includes an **Activity** column with an ASCII histogram (`#` bars) showing each group's count proportional to the largest displayed group. Visual sparkline at a glance.

---

## What's new

### Activity column (default on)
- Fixed `width=22`, `no_wrap=True` so 20-char bars always fit
- Bar length = `max(1, round(count / max_count * 20))`
- Normalized to the **largest displayed group** (respects `--limit`)
- Minimum 1 bar for visibility floor

### `--no-bars` (opt-out)
- Suppresses Activity column
- Falls back to v1.7.18 5-column layout
- Useful for narrow terminals

### JSON / CSV unchanged
The histogram is Rich-only — data-format outputs stay structured.

## Authoritative-principle catches

**2 bugs caught and fixed during testing:**

### Lesson #50 logged: Unicode in piped CLI output
First attempt used `"\u2588"` (U+2588 FULL BLOCK) for prettier rendering. **Rich crashed in non-TTY Windows pipes** — cp1252 codec can't encode U+2588. Fix: switch to ASCII `#`.

**Rule**: prefer ASCII over Unicode in CLI output that may be piped, especially on Windows where cp1252 is the default codec for non-TTY destinations.

### Lesson #51 logged: Rich auto-sizing wraps fixed-content columns
First attempt let Rich auto-size the Activity column. The 20-char bar **wrapped across multiple lines** when piped to non-TTY destinations. Test 6 caught it: "Top row bars: 11" instead of expected 20.

**Rule**: Rich columns with fixed-content (histograms, sparklines, ASCII-art) should always use explicit `width=` + `no_wrap=True`.

### Test design catch
Subprocess output on Windows includes `\r\n` line endings even when `csv.writer(lineterminator="\n")` is set, because Python's `sys.stdout` defaults to text mode. Test 5 failed initially; fixed with `.rstrip("\r")`.

## Limitations

- **ASCII `#` only** — could detect TTY via `sys.stdout.isatty()` and use unicode (`█`) in interactive mode
- **Fixed BAR_WIDTH=20** — not flag-configurable
- **No magnitude-coloring** — all bars green; top=red/tail=yellow would highlight outliers
- **Linear normalization only** — log-scale would compress heavy skew
