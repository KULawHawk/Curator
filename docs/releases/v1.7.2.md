## Headline

`ForecastService` linear-fits monthly indexing rate from the `files` table and projects when each drive hits 95% / 99% capacity. New `curator forecast` CLI command + `ForecastDialog` (Tools menu) surface days-to-threshold projections per drive.

More importantly: the dialog surfaces the **"already past threshold, cleanup urgent"** signal directly. On the canonical DB, this caught the real situation on first run — C:\ at **99.8% full** (950.5 / 952.8 GB). Not a future-projection toy; a current-state cleanup priority.

---

## What's new

### `ForecastService` (new service in `services/forecast.py`)

```python
class ForecastService:
    def compute_disk_forecast(self, drive_path: str) -> DiskForecast: ...
    def compute_all_drives(self) -> list[DiskForecast]: ...
```

Pure-function `_linear_fit(history)` helper does least-squares math on monthly buckets. Five status states:

| Status | Meaning |
|---|---|
| `fit_ok` | Linear fit succeeded; projection has confidence (R² included) |
| `past_95pct` | Drive already at warning threshold; project to critical only |
| `past_99pct` | Drive already at critical; no projection, cleanup is urgent |
| `insufficient_data` | <2 months of `seen_at` history; can't fit yet |
| `no_growth` | Slope ≤ 0 (deletions outpace additions); projection N/A |

### `curator forecast [drive]` CLI command

```
$ curator forecast

C:\
  Used:     950.5 GB / 952.8 GB  (99.8%)
  Free:       2.3 GB
  Drive is already at 99.8% capacity (>= 99% critical). No projection needed - cleanup is urgent.
  History (1 month(s)):
    2026-05: +86,943 files, +10.74 GB
```

`--json` mode produces machine-readable payload with all fields including ISO-format ETA timestamps.

### `ForecastDialog` GUI

Accessible from **Tools → Drive capacity forecast...**

Per-drive cards with:
- Large color-coded percentage badge (green/yellow/red/gray)
- Used / Total / Free / Fill rate stats
- Projection table: threshold | days from now | ETA date
- Monthly history table (last 6 months max)
- Refresh button to recompute

## Limitations

- **Index size != drive used space.** Curator only knows files it has indexed.
- **Need ≥2 months of `seen_at` history** for linear fit. With 1 month (current canonical state), `insufficient_data` status.
- **No retroactive snapshots.** True forecasting would need historical "DB size at month N" snapshots; we approximate via `files.size` aggregated by `seen_at`.
- **`compute_all_drives` skips removable/optical drives** (per `psutil.disk_partitions(all=False)`).
