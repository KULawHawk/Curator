## Headline

`curator scan-pii` gains `--csv` and `--no-header` flags, mirroring v1.7.19/v1.7.20's audit-summary pattern. PII scan results can now be dumped to CSV for spreadsheet review ‚Äî with **two distinct row shapes** depending on `--show-matches`.

---

## What's new

### `--csv` dual-mode

**Per-match mode** (with `--show-matches`): one row per match
```
source,line,offset,pattern,severity,redacted
./my_docs/notes.txt,1,8,github_pat,high,************************************aaaa
./my_docs/notes.txt,2,57,gitlab_pat,high,**********************xxxx
./my_docs/notes.txt,3,89,ssn,high,*******7777
```

**Per-file mode** (default, no `--show-matches`): one row per file
```
source,match_count,has_high,by_pattern,truncated,error
./my_docs/notes.txt,3,yes,github_pat=1;gitlab_pat=1;ssn=1,no,
./my_docs/config.env,5,yes,aws_access_key_id=1;jwt=2;openai_api_key=2,no,
./my_docs/clean.md,0,no,,no,
```

### `--no-header`
Suppresses the CSV header row for piping or appending. Mirrors v1.7.20's audit-summary flag.

### `by_pattern` encoding
Per-file rows encode the pattern‚Üícount map as `name=count;name=count` in a single CSV cell. Splits easily in Python (`dict(p.split("=") for p in s.split(";"))`) or Excel (`TEXTSPLIT(A2, ";")`). Avoids JSON-in-CSV escaping hell.

### Precedence
`--json` wins over `--csv` if both flags are set.

## Authoritative-principle this turn

**0 bugs caught** ‚Äî clean first-try ship. Lesson #49 (unique anchors in large files) applied: the CSV emit branch used `total_files = len(reports)` as a unique sentinel for `scan-pii`'s Rich branch (this exact line doesn't appear in any other command's branch). The previous lesson-#49 mistake (where v1.7.19's CSV initially landed in the wrong command) was not repeated.

**Design lesson on `by_pattern` encoding**: rejected JSON-in-CSV (`{"github_pat": 1, ...}`) in favor of `name=count;name=count`. Reasons:
1. JSON braces/quotes require CSV-quoted escaping that breaks naive pipeline tools
2. Excel doesn't auto-parse JSON cells in any useful way
3. Semicolon-joined format keeps everything in a single unquoted cell AND parses with one line of stdlib

## Limitations

- **No `--local` for scan-pii** ‚Äî output has no timestamps, so TZ doesn't apply
- **`by_pattern` encoding is non-standard** ‚Äî users wanting JSON should use `--json`
- **No `--csv` for `tier`, `export-clean`, `forecast` yet** ‚Äî could batch as v1.8
- **No streaming output** ‚Äî full report list built in memory before emit
- **Redacted-string CSV safety** ‚Äî stdlib `csv.writer` handles future patterns with commas/quotes automatically
