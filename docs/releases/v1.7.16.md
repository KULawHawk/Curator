# Curator v1.7.16 — Release Notes

**Tagged:** 2026-05-11

---

## Headline

The Tier scan dialog now supports keyboard shortcuts:
- **Enter / Return** → Inspect (same as right-click → Inspect)
- **Delete** → Send to trash (with confirmation; same as right-click → Send to trash)

Completes the v1.7.14 actions story so keyboard-driven users don't need to right-click.

---

## What's new

### Workflow

```
1. Open Tools → Tier scan
2. Pick a recipe, click Scan
3. Arrow-key down to the candidate you want
4. Press Enter to inspect, or Delete to trash
```

No mouse required.

### Implementation

- **`eventFilter(obj, event)` method** on TierDialog listens for `QEvent.KeyPress` on the candidate table
- **Two key handlers:**
  - `_handle_enter_shortcut()` → dispatches Inspect on current row
  - `_handle_delete_shortcut()` → dispatches Send-to-trash on current row
- **Shared helper `_resolve_row_to_file_entity(row)`** extracted from the v1.7.14 context-menu code so keyboard and mouse paths use one implementation
- **Event filter installed** in `_build_ui`: `self._table.installEventFilter(self)`
- **Fall-through preserved** for all other keys (arrow navigation, typeahead search, page-up/down)
- **Scoped to the table** — events from other widgets are ignored

## Authoritative-principle catches

**0 bugs caught** — clean first-try ship.

**Lesson #48 logged**: when adding a second consumer of an existing inline code path (here: keyboard shortcuts on top of v1.7.14's context-menu resolution chain), **extract the shared logic into a helper BEFORE adding the second consumer**. The shared helper is then tested once (out-of-range, valid path, error paths) and both consumers benefit from the same correctness guarantees with no drift risk.

## Limitations / next steps

- **No status-change shortcuts (1/2/3/4)** — would conflict with future typeahead-search; deferred
- **No multi-select bulk shortcuts** — Delete still acts on single current row even with multi-select
- **No accelerator hints in menu items** — could add `QKeySequence.Delete` to context menu actions for proper Qt menu rendering with "(Del)" suffix
